# Refactoring Plan — 2026-03-23

Full workspace audit (26 Rust crates + SvelteKit frontend). Prioritized by impact.

---

## 1. Remove 7 Orphaned Engines [Critical]

**Status:** Not started
**Effort:** Medium | **Impact:** High — removes ~2000 lines dead code, clarifies architecture

Architecture specifies 5 engines (Forge, Code, Harvest, Content, GTM), but the workspace has 12. Seven extras — `finance-engine`, `product-engine`, `growth-engine`, `distro-engine`, `legal-engine`, `support-engine`, `infra-engine` — are stubs or partially implemented, never wired into `main.rs`, and pollute `EngineKind` in `rusvel-core/src/domain.rs`.

API routes exist for all 12 departments via macros in `department.rs`, but 7 are dead endpoints. Frontend has placeholder routes for them too.

### Files to change

- `Cargo.toml` — remove 7 workspace members
- `crates/rusvel-core/src/domain.rs` — remove Finance, Product, Growth, Distribution, Legal, Support, Infra from `EngineKind` enum
- `crates/rusvel-api/src/department.rs` — remove macro expansions for the 7 new depts
- `crates/rusvel-api/src/lib.rs` — remove routes for departments 6–12
- `frontend/src/routes/{finance,product,growth,legal,support,infra}/` — delete stub pages
- Delete crate directories: `crates/{finance,product,growth,distro,legal,support,infra}-engine/`

### Rationale

Scope creep at the type level. Re-add engines one at a time when actually ready to implement.

## 2. Dynamic SvelteKit Route for Department Pages [Quick Win]

**Status:** Not started
**Effort:** Low | **Impact:** Medium — eliminates duplication

`forge/+page.svelte`, `code/+page.svelte`, `content/+page.svelte`, `harvest/+page.svelte`, `gtm/+page.svelte` are identical 30-line files differing only in `dept`, `title`, `icon`, `color`, and `quickActions`.

### Action

Replace 5 static routes with a single `[dept]/+page.svelte` dynamic route + config map. Goes from 5×30 lines to ~1×50 lines.

## 5. Fix Dynamic Tailwind Classes in Frontend

**Status:** Not started
**Effort:** Low | **Impact:** High — prevents runtime styling bugs

`DepartmentPanel.svelte` uses dynamic class names like `` bg-${color}-600/20 ``. Tailwind purges classes at build time — dynamic interpolation produces classes that don't exist in the CSS output.

### Action

Use the `tokens.ts` color map to return full pre-defined class strings, or use inline `style` attributes with CSS variables.

## 6. Replace Department Macros with Generic Handler

**Status:** Not started
**Effort:** Medium | **Impact:** Medium — cleaner API layer

`rusvel-api/src/department.rs` (703 lines) uses heavy macro usage to generate identical handler sets for each department. All departments share the same logic: stream chat via Claude CLI, store config, fetch events.

### Action

Replace the macro with a generic `dept_handler(dept: &str)` function. Register routes in a loop over the 5 department names.
